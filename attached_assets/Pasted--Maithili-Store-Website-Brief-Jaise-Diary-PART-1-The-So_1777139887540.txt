## 🎨 PART 2 — Visual Direction
*(Designer ke liye — exact feel)*

---

**Mood Board — 3 Alfaazon Mein:**
> ***Clean. Warm. Confident.***

---

**Color Philosophy:**

> *"Ek Maithili dupatta dekha hai kabhi?*
> *Uski colors — bold hoti hain, par shor nahi karti.*
> *Wahi chahiye."*

| Element | Color Direction |
|---|---|
| **Primary** | Deep Maithili Red `#C0392B` — identity, strength |
| **Secondary** | Warm Ivory `#FAF3E0` — paper, diary, warmth |
| **Accent** | Turmeric Gold `#F39C12` — energy, youth |
| **Neutral** | Charcoal `#2C3E50` — text, grounding |
| **Background** | Off-white `#FDFCFB` — never pure white |

---

**Typography:**

> *"Jo padha jaaye — wo likhna. Jo dikhaya jaaye — woh dikhana."*

| Use | Font Direction |
|---|---|
| **Headlines** | Bold Serif — presence chahiye |
| **Body** | Clean Sans-serif — readability first |
| **Taglines** | Italic — thoda personal feel |
| **Tool Names** | Monospace — technical credibility |

---

**Layout Philosophy:**

> *"Bazaar mein jaate ho toh kya hota hai?*
> *Pehle overview dikhta hai — kya kya hai.*
> *Phir section mein jaate ho.*
> *Phir ek cheez uthate ho.*
> *Yahi flow chahiye."*

- **No carousels** — sab kuch ek hi baar dikhao
- **Grid-based** — categories clearly separated
- **Breathing space** — har element ke beech gap do
- **Mobile-first** — 70% users phone pe aayenge

## ⚙️ PART 4 — Technical Specifications
*(Developer ke liye — exact)*

---

**Performance Requirements:**

> *"Jo jaldi khule — woh zyaada use hota hai.*
> *Simple physics."*

```
Page Load: < 2 seconds
Mobile Score: 90+ (Lighthouse)
Offline: Core tools work without internet
Images: WebP format, lazy loading
Fonts: Self-hosted (no Google Fonts dependency)
```

**Must-Have Features:**

```
✅ Search — instant, fuzzy search
✅ Dark Mode — developer log maangenge
✅ Keyboard Navigation — accessibility
✅ Bookmark tools — user account free
✅ Recently Used — local storage
✅ Share tool — direct link
✅ Copy output — one click
✅ Download output — where applicable
✅ Mobile responsive — every single tool
```

**Tool Organization System:**

```
Each tool has:
├── Unique slug (URL)
├── Category (primary)
├── Tags (multiple)
├── User type (who uses it)
├── Complexity level (beginner/advanced)
├── Free/paid flag
└── Last updated date
```

**Search Intelligence:**

> *"Search mein 'color' likho —*
> *Color picker, Color palette generator,*
> *Color contrast checker —*
> *Teeno dikhne chahiye.*
> *User ko nahi pata exact naam."*

```
Search should match:
├── Tool name
├── Tool description  
├── Tags
├── Category
└── Common synonyms
    (e.g., "resume" = "CV" = "biodata")
```

## 🎯 PART 5 — UX Rules
*(Jo Abir ne khud observe kiya)*

---

> *"Maine notice kiya hai —*
> *Log 3 cheezon se bhaagte hain website pe:*
> *Popup, registration, aur confusion.*
> *Teeno remove karo."*

**Abir's 7 UX Laws for Maithili Store:**

**Law 1:** *"Pehle tool, phir account"*
No forced login. Tool use karo — tab bolo account banao.

**Law 2:** *"Ek kaam, ek page"*
Tool page pe sirf tool. No ads between tool and user.

**Law 3:** *"Error — insaan ki bhasha mein"*
"Something went wrong" nahi —
"Yeh file too large hai. 5MB se kam karo."

**Law 4:** *"Mobile mein bhi poora kaam ho"*
Koi tool mobile mein half-broken nahi hoga.

**Law 5:** *"Speed = Respect"*
Agar tool 5 second se zyaada leta hai —
Loading animation + reason batao.

**Law 6:** *"Empty state — helpful ho"*
Agar koi tool use nahi kiya —
"Yahan se shuru karo →" suggestion do.

**Law 7:** *"Feedback loop — close karo"*
Har tool ke neeche —
"Kaam aaya? 👍 Nahi aaya? 👎"
One click. No form.

---

## 📱 PART 6 — Mobile-First Rules

```
Navigation: Bottom tab bar on mobile
Search: Always accessible — sticky top
Tool Interface: Full screen on mobile
Output: Copy button — big, thumb-friendly
Scroll: Vertical only — no horizontal hell
Font size: Minimum 16px — no squinting
Tap targets: Minimum 44x44px
```
